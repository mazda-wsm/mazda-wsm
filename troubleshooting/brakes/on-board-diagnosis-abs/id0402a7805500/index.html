On-Board Diagnostic (OBD) Test Description

â¢Â The OBD test inspects the integrity and function of the ABS and outputs the results when requested by the specific tests.

â¢Â On-board diagnostic test also:

  • Provides a quick inspection of the ABS usually performed at the start of each diagnostic procedure. {: #wp1059817}- Provides verification after repairs to ensure that no other faults occurred during service.

â¢Â The OBD test is divided into 3 tests:

  • Read/clear diagnostic results, PID monitor and record and active command modes.

Read/clear diagnostic results

â¢Â This function allows you to read or clear DTCs in the ABS HU/CM memory.

PID/Data monitor and record

â¢Â This function allows you to access certain data values, input signals, calculated values, and system status information.

Active command modes

â¢Â This function allows you to control devices through the M-MDS.

Reading DTCs Procedure

  1. Connect the M-MDS to the vehicle DLC-2 connector.{: #wp1059956}

  1. After the vehicle is identified, select the following items from the initial screen of the M-MDS.

â¢Â When using the IDS (laptop PC)

  1. Select the "Toolbox" tab. {: #wp1060010}2. Select "Self Test". {: #wp1060028}3. Select "Modules". {: #wp1060042}4. Select "ABS".

â¢Â When using the PDS (Pocket PC)

  1. Select "Module Tests". {: #wp1060080}2. Select "ABS". {: #wp1060098}3. Select "Self Test".

  2. Verify the DTC according to the directions on the screen.

â¢Â If any DTCs are displayed, perform troubleshooting according to the corresponding DTC inspection.

  1. After completion of repairs, clear all DTCs stored in the ABS. (See Clearing DTCs Procedures.)

Clearing DTCs Procedures

  1. Connect the M-MDS to the vehicle DLC-2 connector.{: #wp1060178}

  1. After the vehicle is identified, select the following items from the initial screen of the M-MDS.

â¢Â When using the IDS (laptop PC)

  1. Select the "Toolbox" tab. {: #wp1060232}2. Select "Self Test". {: #wp1060250}3. Select "Modules". {: #wp1060264}4. Select "ABS".

â¢Â When using the PDS (Pocket PC)

  1. Select "Module Tests". {: #wp1060302}2. Select "ABS". {: #wp1060320}3. Select "Self Test".

  2. Verify the DTC according to the directions on the screen.

  3. Press the clear button on the DTC screen to clear the DTC.

  4. Verify that no DTCs are displayed.

PID/Data Monitor and Record Procedure

  1. Connect the M-MDS to the vehicle DLC-2 connector.{: #wp1060390}

  1. After the vehicle is identified, select the following items from the initial screen of the M-MDS.

â¢Â When using the IDS (laptop PC)

  1. Select the "Toolbox" tab. {: #wp1060444}2. Select "DataLogger". {: #wp1060462}3. Select "Modules". {: #wp1060476}4. Select "ABS".

â¢Â When using the PDS (Pocket PC)

  1. Select "Module Tests". {: #wp1060514}2. Select "ABS". {: #wp1060532}3. Select "DataLogger".

  2. Select the applicable PID from the PID table.

  3. Verify the PID data according to the directions on the screen.

*Note*{: #wp1060580}

â¢Â  The PID/Data monitor function is used for monitoring the calculated value. Therefore, if the monitored value of the output parts is not within the specification, inspection of the monitored value of input parts corresponding to applicable output part control is necessary. In addition, because the system does not display output part malfunction as abnormality in the monitored value, it is necessary to inspect the output part individually using a active command modes function.

Active Command Modes Procedure

  1. Connect the M-MDS to the vehicle DLC-2 connector.{: #wp1060610}

  1. After the vehicle is identified, select the following items from the initial screen of the M-MDS.

â¢Â When using the IDS (laptop PC)

  1. Select the "Toolbox" tab. {: #wp1060674}2. Select "DataLogger". {: #wp1060692}3. Select "Modules". {: #wp1060706}4. Select "ABS".

â¢Â When using the PDS (Pocket PC)

  1. Select "Module Tests". {: #wp1060744}2. Select "ABS". {: #wp1060762}3. Select "DataLogger".

  2. Select the active command modes from the PID table.

  3. Perform the active command modes, inspect the operations for each parts.

â¢Â If there is no operation sound from the relay, motor, and solenoid after the active command mode inspection is performed, it is possible that there is an open or short circuit in the wiring harness, relay, motor or solenoid, or sticking and operation malfunction.
